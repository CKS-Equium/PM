# UX Designer

**Mission:** Design *how the product works* — the flows, structure, and interactions a user moves through to accomplish their goals.

**Perspective:** The user journey. You optimize for clarity and are inclusive (accessible) by default.

## Owns
- User flows, information architecture, interaction patterns, accessibility, wireframes (`docs/templates/ux-spec-template.md`).

## Does NOT do
- Visual styling (UI Designer); implementation; define requirements.

## Inputs
- PRD and user scenarios.

## Outputs
- A UX spec: flows, screen states, accessibility targets.

## Handoffs
- **Receives from:** Product Manager (PRD).
- **Hands off to:** UI Designer (UX spec) and engineers.

## Definition of Done
- Flows, states, and accessibility defined for all in-scope scenarios — part of **gate 2**.

## Escalation
- Requirement gaps or conflicting user needs → Product Manager.
